# 参考文献著录格式速查（供 references 抽取）

本文件用于帮助在 MinerU 解析产物（Markdown）中更稳地定位“参考文献区块”并切分条目。目标不是完美 BibTeX 解析，而是产出可审计的 `raw` + 尽力补全的结构化字段，并用 `confidence` 显式表达不确定性。

本文件基于“理工科 + 中英混排”常见体例整理，重点覆盖：
- 顺序编码制（IEEE/ACM/GB/T 7714 常见）
- 著者‑出版年制（ASCE/APA/Harvard 常见）
- GB/T 7714‑2015 的关键特征（作者截断、“等/et al.”、资源类型标识与载体、电子资源的 DOI/引用日期要求等）

## 1) 常见 references 区块标题

优先按标题行匹配（大小写/语言均考虑）：
- `References` / `REFERENCES`
- `Bibliography`
- `Works Cited`
- `参考文献`
- `引用文献`

多候选时：默认选择“最后一个且长度合理”的区块（正文末尾更常见）。

## 2) 引用体系与风格线索（用于判断切分/抽取策略）

### 2.1 顺序编码制（Numeric System）

典型特征：
- 文后条目通常带编号：`[1] ...` 或 `1. ...`
- IEEE/ACM/GB/T 7714 常见

### 2.2 著者‑出版年制（Author‑Date System）

典型特征：
- 条目行首直接以作者开头，并在作者后较早位置出现年份：`(2020)` 或 `2020`
- ASCE/APA/Harvard 常见
- 常见悬挂缩进：续行前置空格/制表符（同一条目跨行）

### 2.3 若需要“风格猜测”（可选）

可用弱启发式（仅用于调参，不要强依赖）：
- IEEE：作者形如 `J. K. Author`（缩写点不可省略），多作者可能列出前 6 个后 `et al.`
- ACM：更可能使用全名；且 DOI 更常以 `https://doi.org/...` 的完整链接出现
- GB/T 7714：常见“超过 3 人 → 前 3 人 + 等/et al.”；可能出现 `\[J/OL\]` 等“文献类型/载体”标识（示例见下）
- ASCE：作者‑年制；机构标准/工程报告引用中常出现标准号与出版地/出版社（例如 `GB 50010-2010 (2010) ... Beijing.`）

## 3) 常见条目编号模式（强信号）

条目通常以行首编号起始（可跨行）：
- `[1] ...`
- `1. ...`
- `1) ...`

切分策略（启发式）：
- “新条目” = 发现新的编号行首模式
- “条目续行” = 直到下一个编号出现前的所有行

## 4) 非编号型（弱信号，需更谨慎）

常见于 APA/作者‑年份风格：
- `Author, A. A., & Author, B. B. (2020). Title...`
- `Author A, Author B. 2020. Title...`

常用信号（组合判断，避免误切）：
- 行首出现作者串（逗号/and/&/et al. 等）
- 紧随出现 `(19|20)\d{2}` 或 `(?:19|20)\d{2}`
- 行内出现句点分隔的“作者—年份—标题—容器信息”结构

若无法稳定切分：宁可输出少量条目（或单条 `raw`），并降低 `confidence`。

## 5) GB/T 7714‑2015 相关信号（中英混排常见）

### 5.1 作者截断与“等/et al.”

常见规范（用于理解 `raw`，不强制重写）：
- 责任者 ≤ 3：通常全部列出
- 责任者 > 3：常见“仅列前 3 + `等` / `et al.`”

抽取建议：
- `author[]` 只收集 `raw` 中实际出现的作者（不要臆造未出现的作者）
- 若出现 `等` / `et al.`：可视为“作者被截断”的信号，适当降低 `confidence`

### 5.2 文献类型标识/载体标识（强结构信号）

GB/T 7714 常见写法：`\[文献类型标识/载体标识\]`，例如联机网络期刊文章：`\[J/OL\]`。

解析建议：
- 若条目中出现类似 `\[J/OL\]`、`\[J\]`、`\[M\]`、`\[D\]` 等标识：
  - 可作为 `itemType` 的弱线索（不确定则不输出）
  - 标识周边往往接近“标题结束位置”，可辅助 `title` 抽取

常见类型标识（不完整，按需扩展；仅作识别提示）：
- `J` 期刊文章（journal）
- `C` 会议论文集/会议文集（conference proceedings）
- `M` 专著（book）
- `D` 学位论文（thesis/dissertation）
- `R` 报告（report）
- `S` 标准（standard）
- `P` 专利（patent）
- `N` 报纸（newspaper）
- `A` 档案（archive）
- `DS` 数据集（dataset）
- `Z` 其他（other）

载体标识常见：`OL`（online）。

注意：不要把中括号里的类型标识误判为“编号”。

### 5.3 非英语文献的常见附注

在国际期刊中引用中文/非英语文献时，常见形态：
- 仅英文翻译标题，末尾标注 `(in Chinese)`
- 原文标题 + `[...]` 内英文翻译
- 拼音转写（Transliteration）

抽取建议：
- 若出现 `(in Chinese)` 等附注：优先视为“语言说明”，避免并入 `title` 核心语义（但保留在 `raw`）

## 6) MinerU 常见噪声与修复建议

- 断行：同一条目被拆成多行，且中间夹杂多余空格/连字符断词
- 编号丢失：编号被识别到上一行末尾或单独一行
- 页眉页脚污染：出现重复页码/Running title

策略：
- 合并条目时清理多余空白，但不要丢失信息（`raw` 必须可追溯）
- 对可疑页眉页脚（短行、重复出现）保持保守：不强行删除，避免误删正文

## 7) 字段抽取优先级（尽量对齐 Zotero）

强信号优先：
- `DOI`：
  - 裸 DOI：`10.\d{4,9}/...`
  - DOI URL（ACM 常见）：`https://doi.org/10....`
- `url`：`https?://...`
- `arxiv`：`arXiv:\d{4}\.\d{4,5}` 或 `arxiv.org/abs/...`
- `year`：优先第一个可信的 `19xx/20xx`（靠近作者段更可信）

弱信号（允许失败）：
- `author[]` / `title`
- `publicationTitle` / `volume` / `issue` / `pages`
- `itemType` / `creators` / `date`

不确定时的输出策略：
- 永远保留 `raw`
- `author=[]`、`title=""`、`year=null`
- `confidence` 设低值（例如 0.1~0.3）

## 8) 置信度建议（仅供标尺）

可以用“抽取到的字段数量 + 一致性”作为粗略标尺：
- 0.1：仅能保留 `raw`（条目切分或字段边界高度不确定）
- 0.3：能抽取 `year` + (`DOI` 或 `url` 或 `arxiv`) 之一，但 `author/title` 不稳
- 0.5：能抽取 `author/title/year` 中至少两项，且无明显冲突
- 0.7：能抽取 `author/title/year` + 关键标识（`DOI`/`url`/`arxiv`）至少一项

避免“高置信度 + 关键字段为空”的组合；宁可降低 `confidence` 也不要臆造字段。
