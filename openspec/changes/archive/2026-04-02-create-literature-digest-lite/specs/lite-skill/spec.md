## ADDED Requirements

### Requirement: 输入契约
literature-digest-lite SHALL read input parameters from the prompt payload, including `source_path` and `language`.

#### Scenario: 读取 source_path
- **WHEN** agent 调用 skill 时提供 source_path
- **THEN** skill 使用该路径作为唯一内容来源

#### Scenario: source_path 支持多种格式
- **WHEN** source_path 指向 Markdown、PDF、单文件.tex、LaTeX 工程目录或无扩展名文件
- **THEN** skill 能够正确识别并处理多种合法形式的 source_path 并归一化为 source.md

#### Scenario: 读取 language 参数
- **WHEN** agent 调用 skill 时提供 language 参数
- **THEN** skill 使用该参数控制 digest 输出语言

#### Scenario: language 回退逻辑
- **WHEN** language 参数未显式给出或为空
- **THEN** skill 先从 prompt 主要语言推断，仅在无法稳定判断时回退 zh-CN

### Requirement: 输出契约
literature-digest-lite MUST emit a stdout JSON result that contains `digest_path` and provenance information.

#### Scenario: 成功态输出
- **WHEN** digest 生成成功
- **THEN** stdout 输出包含 digest_path、provenance.generated_at、provenance.input_hash、warnings（空数组）、error（null）的 JSON 对象

#### Scenario: 失败态输出
- **WHEN** digest 生成失败（如文件不存在、格式无法识别）
- **THEN** stdout 输出包含 digest_path（空字符串）、provenance（空对象）、warnings（空数组）、error（包含 code 和 message）的 JSON 对象

#### Scenario: 最终产物
- **WHEN** skill 执行完成
- **THEN** 在 output_dir 生成 digest.md 文件

### Requirement: 输入标准化
The skill SHALL provide a normalize mode that reads the input source and performs format detection and normalization.

#### Scenario: PDF 文件处理
- **WHEN** 输入文件是 PDF 格式（文件头命中%PDF-）
- **THEN** 优先使用 pymupdf4llm 转换为 Markdown，失败则使用兜底解析

#### Scenario: LaTeX 单文件处理
- **WHEN** 输入文件扩展名为.tex 或内容命中\documentclass / \begin{document}
- **THEN** 将原文直接复制到 source.md 的 ```tex fence 中

#### Scenario: LaTeX 工程目录处理
- **WHEN** 输入是目录且包含.tex 文件
- **THEN** 自动检测主入口.tex，展平\input / \include 后整体包进 ```tex fence

#### Scenario: Markdown/文本处理
- **WHEN** 输入文件是 Markdown 或纯文本
- **THEN** 直接读取内容

#### Scenario: 计算 input_hash
- **WHEN** 标准化完成后
- **THEN** 计算标准化内容的 sha256 哈希值

### Requirement: Digest 生成
Digest content MUST be generated by the LLM, and the script MUST NOT perform digest-generation reasoning itself.

#### Scenario: digest_slots 结构
- **WHEN** LLM 生成 digest 内容
- **THEN** 必须输出包含 tldr、research_question_and_contributions、method_highlights、key_results、limitations_and_reproducibility 五个固定槽位的结构化数据

#### Scenario: section_summaries 结构
- **WHEN** LLM 生成分章节总结
- **THEN** 必须按大纲顺序提供 source_heading 和 items 列表

### Requirement: Digest 输出结构（zh-CN）
When `language` is `zh-CN`, the final `digest.md` SHALL use the fixed Chinese section headings.

#### Scenario: TL;DR 章节
- **WHEN** language 为 zh-CN
- **THEN** digest.md 必须包含 `## TL;DR` 章节，建议 8~15 行，覆盖问题/方法/结果/局限与可复现性线索

#### Scenario: 研究问题与贡献章节
- **WHEN** language 为 zh-CN
- **THEN** digest.md 必须包含 `## 研究问题与贡献` 章节

#### Scenario: 方法要点章节
- **WHEN** language 为 zh-CN
- **THEN** digest.md 必须包含 `## 方法要点` 章节

#### Scenario: 关键结果章节
- **WHEN** language 为 zh-CN
- **THEN** digest.md 必须包含 `## 关键结果` 章节

#### Scenario: 局限与可复现性线索章节
- **WHEN** language 为 zh-CN
- **THEN** digest.md 必须包含 `## 局限与可复现性线索` 章节

#### Scenario: 分章节总结章节
- **WHEN** language 为 zh-CN
- **THEN** digest.md 必须包含 `## 分章节总结` 章节，至少输出 8 个章节/小节块

### Requirement: Digest 输出结构（en-US）
When `language` is `en-US`, the final `digest.md` SHALL use the fixed English section headings.

#### Scenario: TL;DR 章节
- **WHEN** language 为 en-US
- **THEN** digest.md 必须包含 `## TL;DR` 章节，prefer 8–15 lines

#### Scenario: Research Question & Contributions 章节
- **WHEN** language 为 en-US
- **THEN** digest.md 必须包含 `## Research Question & Contributions` 章节

#### Scenario: Method Highlights 章节
- **WHEN** language 为 en-US
- **THEN** digest.md 必须包含 `## Method Highlights` 章节

#### Scenario: Key Results 章节
- **WHEN** language 为 en-US
- **THEN** digest.md 必须包含 `## Key Results` 章节

#### Scenario: Limitations & Reproducibility 章节
- **WHEN** language 为 en-US
- **THEN** digest.md 必须包含 `## Limitations & Reproducibility` 章节

#### Scenario: Section-by-Section Summary 章节
- **WHEN** language 为 en-US
- **THEN** digest.md 必须包含 `## Section-by-Section Summary` 章节，至少输出 8 个章节/小节块

### Requirement: 模板翻译
When `language` is neither `en-*` nor `zh-*`, the agent SHALL translate the template's fixed heading text.

#### Scenario: en-* 语言模板选择
- **WHEN** language.lower().startswith("en")
- **THEN** 直接复制 digest.en-US.md.j2 作为运行时模板

#### Scenario: zh-* 语言模板选择
- **WHEN** language.lower().startswith("zh")
- **THEN** 直接复制 digest.zh-CN.md.j2 作为运行时模板

#### Scenario: 其他语言模板翻译
- **WHEN** language 不以 en 或 zh 开头
- **THEN** agent 直接翻译 digest.zh-CN.md.j2 的固定标题文本，保留 Jinja2 语法

#### Scenario: 翻译原则
- **WHEN** agent 翻译模板
- **THEN** 仅翻译固定标题文本，保留{% ... %}、{{ ... }}、{# ... #}语法，保留变量名和循环结构

### Requirement: 模板渲染
The skill SHALL provide a render mode that renders the final Markdown from structured digest data.

#### Scenario: 读取 payload
- **WHEN** render 模式启动
- **THEN** 从--payload-file 或 stdin 读取包含 digest_slots + section_summaries 的 JSON

#### Scenario: 选择模板
- **WHEN** 提供--template-file 参数
- **THEN** 使用该路径的模板；否则根据 language 选择仓库模板

#### Scenario: 渲染 digest
- **WHEN** 模板和 payload 准备完成
- **THEN** 使用 Jinja2 渲染 digest.md，输出目录为 source_path 所在目录

### Requirement: LLM 与脚本职责边界
The skill MUST preserve a clear responsibility boundary between the LLM and the script runtime.

#### Scenario: LLM 职责
- **WHEN** 执行 digest 生成
- **THEN** digest 槽位内容生成和模板翻译必须由 LLM 完成

#### Scenario: 脚本职责
- **WHEN** 执行 digest 生成
- **THEN** 输入协议探测与标准化、模板渲染、输出 JSON 合法性检查必须由脚本完成

#### Scenario: 绝对禁止
- **WHEN** 执行 digest 生成
- **THEN** 禁止用脚本替代 LLM 做摘要、大纲、语义分类，禁止脚本调用外部 LLM API

### Requirement: 分章节总结细则
Section-by-section summaries MUST satisfy the required granularity and minimum coverage requirements.

#### Scenario: 优先按原文大纲
- **WHEN** 可以可靠识别原文大纲
- **THEN** 优先按"全文大纲骨架"的章节标题逐章总结，推荐使用 `### <原文章节标题>`

#### Scenario: 章节粒度
- **WHEN** 某一级章节过长或包含多个子主题
- **THEN** 优先进一步拆成二级小节 `#### <子节标题或子主题>`

#### Scenario: 数量要求
- **WHEN** 生成分章节总结
- **THEN** 至少输出 8 个章节/小节块（### 或####），并尽量更多

#### Scenario: 内容要求
- **WHEN** 生成分章节总结
- **THEN** 总输出量应显著增加，包含更细粒度的术语/变量/设置/结论/边界条件

#### Scenario: 退化策略
- **WHEN** 无法可靠识别大纲
- **THEN** 退化为 `### 片段 1/2/3...` 的分段总结，片段数量至少 8 段
